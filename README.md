# AI Engineer Learning Roadmap

A hands-on, project-based learning path to become an AI Engineer.

## Your Current Knowledge
- AI vs AGI concepts
- What is an Agent
- MCP (Model Context Protocol) basics
- Basic Prompt Engineering
- Basic Neural Networks

## Learning Path Overview

| Phase | Duration | Focus | Project |
|-------|----------|-------|---------|
| 1 | 2-3 weeks | Foundations | AI Chatbot CLI |
| 2 | 4-6 weeks | Core AI Engineering | Document Q&A (RAG) |
| 3 | 4-6 weeks | Agents & Automation | AI Research Agent |
| 4 | 4-6 weeks | Production AI | Full AI SaaS App |

**Total Estimated Time: 14-21 weeks**

## Projects

### [Project 1: AI Chatbot CLI](./projects/01-ai-chatbot-cli/)
Build a command-line chatbot with LLM integration, conversation history, and advanced prompting techniques.

| Steps | Topics |
|-------|--------|
| 8 steps | Python, APIs, Prompt Engineering, Streaming, System Prompts |

**Key Learning**: How to interact with LLM APIs, manage conversations, and engineer prompts.

---

### [Project 2: Document Q&A (RAG)](./projects/02-document-qa-rag/)
Create a system that answers questions from your documents using RAG architecture.

| Steps | Topics |
|-------|--------|
| 9 steps | Embeddings, Vector Databases, Chunking, Semantic Search, RAG Pipeline |

**Key Learning**: How to make LLMs answer questions from your own data.

---

### [Project 3: AI Research Agent](./projects/03-ai-research-agent/)
Build an autonomous agent that researches topics using tools and MCP.

| Steps | Topics |
|-------|--------|
| 10 steps | AI Agents, ReAct Pattern, Function Calling, MCP Protocol, Tool Building |

**Key Learning**: How to build AI that can take actions and use tools autonomously.

---

### [Project 4: Full AI SaaS Application](./projects/04-ai-saas-app/)
Deploy a production-ready AI application with authentication, billing, and monitoring.

| Steps | Topics |
|-------|--------|
| 12 steps | FastAPI, Databases, Auth, Rate Limiting, Docker, Deployment, Monitoring |

**Key Learning**: How to build and deploy a real AI product.

## How Each Project README is Structured

Every project README contains:

1. **Overview** - What you're building
2. **Prerequisites** - What you need to know first
3. **Learning Objectives** - Checkboxes to track progress
4. **Step-by-Step Guide** - Each step includes:
   - Tasks to complete
   - Documentation links to read
   - What to implement
   - Checkpoint to verify success
5. **Self-Check Questions** - Test your understanding
6. **Stretch Goals** - Optional advanced features

## Tips for Learning

1. **Read the docs first** - Each step links to official documentation
2. **Implement yourself** - Don't copy code; understand and write it
3. **Use checkpoints** - Verify each step works before moving on
4. **Take notes** - Each README has a notes section for your learnings
5. **Ask questions** - When stuck, understand WHY something works
